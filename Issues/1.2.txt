### Analysis of the Bug

The current implementation suffers from severe type ambiguity and a hidden math bug:

1. **Type Schizophrenia**: `up_bytes` and `down_bytes` are initialized as `[0]` (lists), but the system frequently overwrites or checks if they are integers. This forces the codebase to rely on messy `isinstance()` checks at runtime.
2. **The Double-Counting Math Bug**: In `telegram.py`, `transfer_obj.up_bytes[0]` is actively mutated to track the `current` progress of an upload. When the upload completes successfully, the code does `transfer_obj.up_bytes.append(file_size)`. This means the list looks like `[file_size, file_size]`, effectively double-counting the bytes and causing `sum(up_bytes)` to report 200% of the actual transferred data.
3. **Retry Corruption**: If a Pyrogram upload fails midway and retries, `current` restarts at `0`. In a purely additive list/integer setup, this corrupts the aggregate total.

### The Solution: Strict Continuous Integer via Delta Tracking

We will enforce a strict, single data type: **Continuous Integer (`int`)**.

To make this mathematically safe without using lists, we will use a **Delta Tracking System**. Inside Pyrogram's `up_progress` callback, we will calculate the difference (`delta`) between the new `current` bytes and the `last_current` bytes. We add this delta to the continuous integer.

* **Why this is safe:** If an upload fails midway and retries, we simply subtract `last_current` from our total integer before restarting the loop. This guarantees that `up_bytes` perfectly reflects the cumulative state of the entire task, exactly as required for health metrics, without ever mixing types.

Here is the refactored code for all three files.

### 2. `colab_leecher/utility/task_context.py`

We clean up `TaskTransfer` to default to `0`, removing all `isinstance()` and `sum()` overhead, especially in `remove_task`.

```python
@dataclass
class TaskTransfer:
    """Per-task transfer statistics"""
    down_bytes: int = 0
    up_bytes: int = 0
    total_size: int = 0  
    sent_file: List = field(default_factory=list)
    sent_file_names: List[str] = field(default_factory=list)
    successful_downloads: List[dict] = field(default_factory=list)
    start_time: float = field(default_factory=lambda: time.time())
    last_speed: str = "0 B/s"
    last_speed_bytes: float = 0.0

    def reset(self):
        """Reset transfer statistics"""
        self.down_bytes = 0
        self.up_bytes = 0
        self.total_size = 0
        self.sent_file = []
        self.sent_file_names = []
        self.successful_downloads = []
        self.start_time = time.time()
        self.last_speed = "0 B/s"

    def get_current_bytes(self) -> int:
        """Get current transferred bytes (max of down or up)"""
        return max(self.down_bytes, self.up_bytes)
    
    # ... keep get_percentage, get_eta, get_speed identical ...

```

**Update `TaskQueue.remove_task` method:**

```python
    async def remove_task(self, task_id: str) -> Optional[TaskContext]:
        """Remove a completed/cancelled task (thread-safe)"""
        async with self._lock:
            task_ctx = self.active_tasks.pop(task_id, None)
            if task_ctx:
                self.metrics.active_tasks = len(self.active_tasks)

                duration = task_ctx.get_elapsed_time()
                if task_ctx.is_cancelled:
                    self.metrics.record_cancellation()
                elif task_ctx.error.state:
                    self.metrics.record_failure()
                else:
                    self.metrics.record_completion(duration)

                # Track data volume strictly as integers (no lists or isinstance checks)
                self.metrics.total_bytes_downloaded += task_ctx.transfer.down_bytes
                self.metrics.total_bytes_uploaded += task_ctx.transfer.up_bytes

                log.info(
                    f"Task {task_ctx.get_short_id()} removed from queue. Remaining: {len(self.active_tasks)}"
                )
            return task_ctx
```

### 3. `colab_leecher/uploader/telegram.py`

We implement the `last_current` delta tracker and failure-rollback mechanism inside `upload_file`.

```python
async def upload_file(file_path: str, display_name: str, task_ctx: TaskContext = None) -> bool:
    # ... (Keep initialization and zero-byte file checks exactly the same) ...

    # NEW: Set total size for dashboard tracking
    # We NO LONGER mutate or reset `up_bytes` to 0 here. It acts as a continuous cumulative integer.
    if transfer_obj:
        transfer_obj.total_size = file_size

    # ... (Keep all thumbnail, dimension, and caption logic exactly the same) ...

    # --- Initialize Upload State Variables ---
    upload_success = False
    sent_message = None
    last_progress_time = 0
    retry_count = 0
    max_retries = 4 
    error_reason = "Upload Failed"
    upload_start_time = 0 
    
    # Delta tracker for safe continuous integer arithmetic
    last_current = 0 

    # --- Define Progress Callback ---
    async def up_progress(current, total):
        nonlocal last_progress_time, upload_start_time, base_upload_name, last_current 
        now = time.time()
        
        # Calculate delta and apply to the continuous integer
        delta = current - last_current
        last_current = current
        
        if transfer_obj:
            transfer_obj.up_bytes += delta
            
        if now - last_progress_time > 2.5: 
            last_progress_time = now
            try:
                speed_string, eta_seconds, percentage = helper.speedETA(upload_start_time, current, total)
                
                if transfer_obj:
                    transfer_obj.last_speed = speed_string
                    try:
                        transfer_obj.last_speed_bytes = current / (now - upload_start_time) if (now - upload_start_time) > 0 else 0
                    except: pass
                
                done_str = helper.sizeUnit(current)
                total_str = helper.sizeUnit(total)
                eta_str = helper.getTime(eta_seconds)

                status_head = f"<b>Uploading{' ' + task_id_str if task_ctx else ''}</b>\n\n<b>Name:</b> <code>{escape(base_upload_name)}</code>\n"
                await helper.status_bar(status_head, speed_string, percentage, eta_str, done_str, total_str, "TG Upload 🚀", task_ctx=task_ctx)
            except Exception as progress_err:
                log.warning(f"Error updating progress bar: {progress_err}")

    # --- Start Upload Attempt Loop ---
    while retry_count <= max_retries:
        try:
            upload_start_time = time.time() 
            last_progress_time = 0 
            
            # CRITICAL: If Pyrogram retry triggered, we rollback the corrupted partial bytes 
            # from the last failed attempt using our `last_current` state before restarting.
            if transfer_obj and last_current > 0:
                transfer_obj.up_bytes -= last_current
            last_current = 0

            # ... (Keep Pyrogram bot.send_video / send_photo / send_document blocks exactly the same) ...

            if sent_message:
                log.info(f"Successfully uploaded {task_id_str} '{base_upload_name}' (File: {actual_upload_filename}) to chat {target_chat_id} - Msg ID: {sent_message.id}")
                try:
                    transfer_obj.sent_file.append(sent_message)
                    transfer_obj.sent_file_names.append(base_upload_name)
                except AttributeError:
                    log.warning(f"Could not record sent file details {task_id_str}, transfer object might be missing attributes.")
                upload_success = True
                break 
            
            # ... (Keep retry exceptions blocks exactly the same) ...
            
    # --- After Retry Loop ---
    if not upload_success:
        log.error(f"Final upload status {task_id_str} for {actual_upload_filename}: FAILED. Reason: {error_reason}")
        failed_info = {"link": "N/A", "filename": base_upload_name, "index": "Upload", "reason": error_reason}
        try:
            if error_obj: error_obj.failed_links.append(failed_info)
        except AttributeError:
             pass
        return False
    else:
        # Final safety catch: if Pyrogram misses the final 100% callback, we append the missed bytes exactly.
        try:
            if transfer_obj:
                missed_bytes = file_size - last_current
                if missed_bytes > 0:
                    transfer_obj.up_bytes += missed_bytes
        except Exception as report_err:
            log.error(f"Error reporting uploaded bytes {task_id_str} for {actual_upload_filename}: {report_err}")
        return True
```
