To answer your question directly: **No, it is not better to use the temporary zip file's size compared to the original file size for the progress percentage.** Here is why reading the `7z` output directly is much better:

1. **Unpredictable Compression Ratios:** Archiving usually compresses files. If you are compressing a 10 GB file and it compresses down to 4 GB, comparing the output size to the input size means your progress bar would slowly tick up to `40%` and then suddenly jump to `100%` when the task finishes. It would never reflect the actual progress accurately.
2. **Accurate Input Processing:** The percentage that `7z` natively reports (via the `-bsp1` flag) is based on **how many bytes of the input file it has processed**. This gives you a smooth, linear, and mathematically accurate progress bar, regardless of how much the data is being compressed.
