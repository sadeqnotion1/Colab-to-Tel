Based on the error logs you provided, there are three distinct issues occurring in the bot:

1. **`TypeError: unsupported operand type(s) for +=: 'int' and 'list'`:**
The TikTok bulk downloader assigns a `list` to `task_ctx.transfer.down_bytes` (and `up_bytes`), but the cleanup function inside `TaskQueue.remove_task()` expects it to be an `int`, causing the bot to crash during task cancellation.
2. **"Session stopped" when changing dashboard pages:**
Pyrogram isn't actually crashing. When `move_to_bottom=True` is triggered, the bot deletes the dashboard and sends a brand-new photo. Pyrogram opens a separate temporary connection to upload media (DC4 media IPv4) and immediately disconnects it when done. However, because it's re-uploading the photo on every single page turn, it makes the dashboard extremely slow to react and spams your logs. We can fix this by setting `move_to_bottom=False` when paging.
3. **7z stuck at 0% (Throttling bug):**
The 7z archiver's percentage progresses very rapidly (`1%`, `2%`, etc.), which means the `percentage > last_reported_pct` condition fires constantly. It sends dozens of updates per second to `status_bar`, causing `status_bar` to aggressively throttle them all (except the 0% one) because `force_update=False`. We need to use a strict time-based throttle inside the 7z loop and enforce the update.

async def force_update_summary(client=None, page: Optional[int] = None):
    """
    Force update summary dashboard with tracked debounce logic.
    Ensures bursts of task updates (e.g., bulk cancellations or fast downloads)
    don't slam the Telegram API with redundant move_to_bottom API calls.
    """
    global _scheduled_update_task, _last_force_time

    now = time.time()
    time_since_last = now - _last_force_time

    # If it's been more than 2 seconds since the last forced update, push immediately
    if time_since_last >= 2.0:
        _last_force_time = now
        
        # If there happens to be a pending delayed update, safely cancel it
        if _scheduled_update_task and not _scheduled_update_task.done():
            _scheduled_update_task.cancel()
            
        await update_summary_dashboard(client, force=True, move_to_bottom=True, page=page)
        
    else:
        # Debouncing: If we are already waiting to push a delayed update, don't queue another
        if _scheduled_update_task and not _scheduled_update_task.done():
            return
            
        async def delayed_update():
            try:
                # Wait for the remainder of the throttle window to allow the burst to settle
                await asyncio.sleep(2.0 - time_since_last)
                global _last_force_time
                _last_force_time = time.time()
                
                await update_summary_dashboard(client, force=True, move_to_bottom=True, page=page)
            except asyncio.CancelledError:
                pass  # Task was interrupted by a new immediate update, safely exit
            except Exception as e:
                log.error(f"Debounced background update failed: {e}")

        # Track the delayed task in the master system to prevent orphaned memory leaks
        _scheduled_update_task = TASK_QUEUE.create_background_task(
            delayed_update(), 
            "debounced_dashboard_update"
        )
