# Audit Workspace

This folder contains Day 1 audit artifacts for the world-class repository program.

## Files
- `charter.md`: Audit charter, objectives, scope, success criteria, approvals.
- `scorecard.md`: 0-5 scoring rubric and scoring worksheet.
- `metric-catalog.md`: Metric definitions, formulas, owners, cadence, targets.
- `scope-and-risk-map.md`: System boundaries, dependency map, risk register.
- `governance-cadence.md`: Meeting cadence, decision rights, escalation matrix.
- `day1-decisions-log.md`: Decision log, open questions, action tracker.
- `baseline.md`: Initial measured baseline and prioritized findings.
- `findings-automated.csv`: Week 2 automated scan findings with severity, owner, and ETA.
- `findings-automated-summary.md`: Human-readable summary of automated findings.
- `findings-manual.md`: Week 2-3 manual deep-audit findings with reproducible evidence and remediation sequencing.
- `release-gates.md`: Week 3 release gate definitions, enforcement sequence, and runbook links.
- `roadmap-90d.md`: Week 4 30/60/90-day remediation roadmap with P0/P1/P2 priorities, owners, and deadlines.
- `complexity-f-baseline.json`: Baseline allowlist for existing F-grade complexity hotspots (used to block new regressions in CI).
- `multi-terminal-checklist.md`: Worktree-based execution checklist for running roadmap jobs concurrently across multiple terminals.

## Raw Outputs
- `audit/raw/`: local scan outputs and derived metadata.
- `audit/raw/ci/`: CI dependency-audit artifacts generated by `dependency-audit-linux`.
- `docs/operations/evidence/`: executed release, rollback drill, and incident tabletop evidence bundles tracked with runbook templates.

## Recommended Day 1 Order
1. Fill `charter.md`
2. Finalize `scorecard.md`
3. Confirm data sources in `metric-catalog.md`
4. Complete `scope-and-risk-map.md`
5. Align operating model in `governance-cadence.md`
6. Capture outcomes in `day1-decisions-log.md`

## Rule
No Day 2 baseline work starts until all Day 1 files are at least 90% complete and approved.
